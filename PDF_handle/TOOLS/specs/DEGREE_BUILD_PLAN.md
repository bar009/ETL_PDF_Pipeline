# Degree Build Plan

## Intent

This plan turns the Phase L architecture into future execution order.
It does not claim that Level2 or Level3 are already built.

## Order

### 1. Metadata normalization

Before any new degree build:

- keep the current runtime stable
- add or stage Phase L metadata during future content work
- normalize how shared-reference entries are marked

Target result:

- each promoted item has an explicit lane owner
- each promoted item has a declared partition role
- placeholders can be reported honestly

### 2. Level2 discovery and boundary spec

Deliverables:

- Level2 scope spec
- Level2 goldset
- approved category set for a real Fellow Craft lane

Do not reuse Level1 assumptions blindly.
Level2 needs its own native boundary.

### 3. Level2 build

Minimum build threshold:

- real category set
- non-placeholder slugs
- populated explanatory fields
- audit that the lane is more than shell content

Only after this threshold should Level2 be described as built.

### 4. Shared-reference normalization

After Level2 exists as a real lane:

- review all shared lodge-role and glossary/reference material
- decide whether each item stays owned by one lane or becomes explicit framed cross-degree reference
- avoid creating a fake shared lane with undefined ownership

### 5. Level3 discovery and boundary spec

Deliverables:

- Level3 scope spec
- Level3 goldset
- approved category set for native degree-3 content

Level3 must remain separate from Royal Arch and appendant systems.

### 6. Level3 build

Minimum build threshold:

- runtime file exists
- degree registry entry exists
- category set is approved
- entries are not just placeholders or shells

Until then, Level3 remains `not_built`.

### 7. Library surfacing and queue promotion

After the degree lanes are truthful:

- map preservation library buckets into stable site-facing library categories
- promote selected future-entry queues only when they have canonical ownership and product fit

## Non-negotiable rule

Do not call a lane built because a file exists.
Call it built only when the content, metadata, and navigation all support a real product claim.
