# Preview Verification Checklist

Use this checklist when validating a UI pass against a live preview.

## Before Opening The Browser

1. confirm the local URL returns `200`
2. confirm source files and preview files match if sync is expected
3. know which homepage and which topic slug you want to inspect

## Homepage Checks

- does the first viewport feel smaller and clearer
- is there one dominant action
- do cards or panels still feel oversized
- does the right-side structure help, or just take space

## Topic Page Checks

- how much chrome appears before the main reading block
- whether the title block feels oversized
- whether support blocks feel like bubbles
- how soon the user reaches the core idea
- whether scrolling feels excessive at 100% zoom

## If The Browser Tool Fails

Fallback checks:

1. confirm the preview path is correct
2. confirm the modified files were copied
3. confirm hashes match
4. avoid overstating visual certainty
